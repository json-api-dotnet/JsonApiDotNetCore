Class ResourceDefinition<TResource>

exposes developer friendly hooks into how their resources are exposed. It is intended to improve the experience and reduce boilerplate for commonly required features. The goal of this class is to reduce the frequency with which developers have to override the service and repository layers.

BeforeImplicitUpdateRelationship(IRelationshipsDictionary<TResource>, ResourcePipeline)

Implement this hook to run custom logic in the DefaultResourceService<TResource> layer just before implicitly updating relationships to entities of type TResource.

This hook is fired when a relationship to entities of type TResource is implicitly affected from a dependent pipeline (Patch or Delete). For example, if an Article was updated by setting its author relationship (one-to-one) to an existing Person, and by this the relationship to a different Person was implicitly removed, this hook will be fired for the latter Person.

See BeforeUpdate(IDiffableEntityHashSet<TResource>, ResourcePipeline) for information about when this hook is fired.

BeforeUpdate(IDiffableEntityHashSet<TResource>, ResourcePipeline)

Implement this hook to run custom logic in the DefaultResourceService<TResource> layer just before updating entities of type TResource.

For the Patch pipeline, the entities will typically contain one entity.

The returned may be a subset of the DiffableEntityHashSet<TResource> property in parameter entities, in which case the operation of the pipeline will not be executed for the omitted entities. The returned set may also contain custom changes of the properties on the entities.

If new relationships are to be created with the to-be-updated entities, this will be reflected by the corresponding NavigationProperty being set. For each of these relationships, the BeforeUpdateRelationship(HashSet<String>, IRelationshipsDictionary<TResource>, ResourcePipeline) hook is fired after the execution of this hook.

If by the creation of these relationships, any other relationships (eg in the case of an already populated one-to-one relationship) are implicitly affected, the BeforeImplicitUpdateRelationship(IRelationshipsDictionary<TResource>, ResourcePipeline) hook is fired for these.

BeforeUpdateRelationship(HashSet<String>, IRelationshipsDictionary<TResource>, ResourcePipeline)

Implement this hook to run custom logic in the DefaultResourceService<TResource> layer just before updating relationships to entities of type TResource.

This hook is fired when a relationship is created to entities of type TResource from a dependent pipeline (Post or Patch). For example, If an Article was created and its author relationship was set to an existing Person, this hook will be fired for that particular Person.

The returned may be a subset of ids, in which case the operation of the pipeline will not be executed for any entity whose id was omitted

GetQueryFilters()

Define a set of custom query expressions that can be applied instead of the default query behavior. A common use-case for this is including related resources and filtering on them.

Declaration
public virtual ResourceDefinition<TResource>.QueryFilters GetQueryFilters()
Returns
Type Description
ResourceDefinition.QueryFilters<>

A set of custom queries that will be applied instead of the default queries for the given key. Null will be returned if default behavior is desired.
Examples
protected override QueryFilters GetQueryFilters() =>  { 
    { "facility", (t, value) => t.Include(t => t.Tenant)
                                  .Where(t => t.Facility == value) }
 }

    => new QueryFilters {
        { "is-active", FilterIsActive }
    };

{
