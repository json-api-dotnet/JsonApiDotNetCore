BeforeCreate(IEntityHashSet<TResource>, ResourcePipeline)

Implement this hook to run custom logic in the EntityResourceService<TResource> layer just before creation of entities of type TResource.

For the Post pipeline, entities will typically contain one entry. For , entities can contain multiple entities.

The returned System.Collections.Generic.IEnumerable<T> may be a subset of entities, in which case the operation of the pipeline will not be executed for the omitted entities. The returned set may also contain custom changes of the properties on the entities.

If new relationships are to be created with the to-be-created entities, this will be reflected by the corresponding NavigationProperty being set. For each of these relationships, the BeforeUpdateRelationship(HashSet<String>, IRelationshipsDictionary<T>, ResourcePipeline) hook is fired after the execution of this hook.

BeforeDelete(IEntityHashSet<TResource>, ResourcePipeline)

Implement this hook to run custom logic in the EntityResourceService<TResource> layer just before deleting entities of type TResource.

For the Delete pipeline, entities will typically contain one entity. For BulkDelete, this it may contain multiple entities.

The returned System.Collections.Generic.IEnumerable<T> may be a subset of entities, in which case the operation of the pipeline will not be executed for the omitted entities.

If by the deletion of these entities any other entities are affected implicitly by the removal of their relationships (eg in the case of an one-to-one relationship), the BeforeImplicitUpdateRelationship(IRelationshipsDictionary<T>, ResourcePipeline) hook is fired for these entities.

BeforeImplicitUpdateRelationship(IRelationshipsDictionary<TResource>, ResourcePipeline)

Implement this hook to run custom logic in the EntityResourceService<TResource> layer just before implicitly updating relationships to entities of type TResource.

This hook is fired when a relationship to entities of type TResource is implicitly affected from a dependent pipeline (Patch or Delete). For example, if an Article was updated by setting its author relationship (one-to-one) to an existing Person, and by this the relationship to a different Person was implicitly removed, this hook will be fired for the latter Person.

See BeforeUpdate(IDiffableEntityHashSet<T>, ResourcePipeline) for information about when this hook is fired.

BeforeUpdate(IDiffableEntityHashSet<TResource>, ResourcePipeline)

Implement this hook to run custom logic in the EntityResourceService<TResource> layer just before updating entities of type TResource.

For the Patch pipeline, the entities will typically contain one entity. For BulkPatch, this it may contain multiple entities.

The returned System.Collections.Generic.IEnumerable<T> may be a subset of the DiffableEntityHashSet<TResource> property in parameter entities, in which case the operation of the pipeline will not be executed for the omitted entities. The returned set may also contain custom changes of the properties on the entities.

If new relationships are to be created with the to-be-updated entities, this will be reflected by the corresponding NavigationProperty being set. For each of these relationships, the BeforeUpdateRelationship(HashSet<String>, IRelationshipsDictionary<T>, ResourcePipeline) hook is fired after the execution of this hook.

If by the creation of these relationships, any other relationships (eg in the case of an already populated one-to-one relationship) are implicitly affected, the BeforeImplicitUpdateRelationship(IRelationshipsDictionary<T>, ResourcePipeline) hook is fired for these.

BeforeUpdateRelationship(HashSet<String>, IRelationshipsDictionary<TResource>, ResourcePipeline)

Implement this hook to run custom logic in the EntityResourceService<TResource> layer just before updating relationships to entities of type TResource.

This hook is fired when a relationship is created to entities of type TResource from a dependent pipeline (Post or Patch). For example, If an Article was created and its author relationship was set to an existing Person, this hook will be fired for that particular Person.

The returned System.Collections.Generic.IEnumerable<T> may be a subset of ids, in which case the operation of the pipeline will not be executed for any entity whose id was omitted
