ASP.NET Model Validation

The code here shows how to use ASP.NET Model Validation attributes.

Tip

See Atomic Operations for how to implement a custom model validator.

The resource types are decorated with Model Validation attributes, such as [Required], [RegularExpression], [MinLength], and [Range].

Only the fields that appear in a request body (partial POST/PATCH) are validated. When validation fails, the source pointer in the response indicates which attribute(s) are invalid.

Model Validation is enabled by default, but can be turned off in options. Aside from adding validation attributes to your resource properties, no further code is needed.
